## 需求摘要

擴展機票搜尋功能：(1) 新增一般的單程和來回機票搜尋（目前只有外站票）；(2) 將可選的機場從 9 個擴展到全球 ~9,000+ 個（使用 IATA 機場資料庫）；(3) 驗證能否從真實航空公司官網取得正確的票價資料，替換目前的模擬資料。

## 使用者故事

### 機票搜尋者（Ticket Searcher）

#### US-12：全球機場搜尋選擇器
- **作為**機票搜尋者，**我想要**在輸入出發地或目的地時，能搜尋並選擇全球任何機場，**以便**不受限於少數幾個城市，可以規劃到任何地方的旅程
- **驗收條件**：
  - [ ] 機場資料庫包含全球 IATA 認證的機場（至少 5,000+）
  - [ ] 輸入框支援模糊搜尋：輸入城市名稱（中文或英文）或機場代碼都能找到
  - [ ] 搜尋結果顯示：機場代碼、城市名稱、國家、機場全名
  - [ ] 支援中文和英文搜尋（例如「東京」或「Tokyo」都能找到 NRT/HND）
  - [ ] 常用機場優先顯示（根據使用頻率或預設熱門清單）
  - [ ] 搜尋回應速度 < 200ms（前端本地搜尋，不需要呼叫 API）

#### US-13：單程機票搜尋
- **作為**機票搜尋者，**我想要**搜尋單程機票（只有去程，沒有回程），**以便**規劃單程旅行或開放式行程
- **驗收條件**：
  - [ ] 搜尋表單可選擇「單程」或「來回」模式
  - [ ] 單程模式只需填：出發地、目的地、出發日期、人數
  - [ ] 搜尋結果顯示各航空公司的單程票價、出發/到達時間、飛行時長
  - [ ] 結果可依價格、出發時間、飛行時長、航空公司排序
  - [ ] 標示直飛和轉機航班，轉機航班顯示轉機城市和等待時間

#### US-14：來回機票搜尋
- **作為**機票搜尋者，**我想要**搜尋來回機票（去程 + 回程），**以便**比較各航空公司的來回票價
- **驗收條件**：
  - [ ] 來回模式需填：出發地、目的地、去程日期、回程日期、人數
  - [ ] 結果顯示去程和回程的航班組合，包含各段時間和票價
  - [ ] 顯示來回總價，可和外站票價格做比較
  - [ ] 結果可依總價、去程時間、回程時間排序
  - [ ] 去程和回程可以是不同航空公司的組合

#### US-15：真實航空公司票價資料
- **作為**機票搜尋者，**我想要**看到的是真實的航空公司票價（而非模擬資料），**以便**做出正確的購票決策
- **驗收條件**：
  - [ ] 至少支援 3 家航空公司的真實票價查詢
  - [ ] 票價資料標示資料來源和查詢時間
  - [ ] 當真實資料查詢失敗時，清楚標示「模擬資料」而非偽裝為真實
  - [ ] 真實資料查詢時間 < 15 秒（可顯示載入進度）
  - [ ] 資料快取機制：同一條件 30 分鐘內不重複查詢

#### US-16：搜尋結果與外站票比較
- **作為**機票搜尋者，**我想要**在搜尋來回機票時，同時看到外站票的方案比較，**以便**一眼看出哪種買法最划算
- **驗收條件**：
  - [ ] 來回搜尋結果頁面底部或側邊顯示「外站票方案」區塊
  - [ ] 顯示外站票最低價 vs 直飛來回最低價的價差
  - [ ] 提供「查看外站票詳情」的連結，跳轉到外站票搜尋

## 衝突標記

- ⚠️ 真實航空公司資料爬取可能面臨反爬蟲機制，導致資料不穩定 → 需要 fallback 到模擬資料，並明確告知使用者
- ⚠️ 全球機場 ~9,000+ 個資料量大，需要考慮前端搜尋效能 → 使用前端本地搜尋 + 延遲載入
- ⚠️ 真實資料查詢時間可能較長（爬蟲需要時間） → UI 需要顯示載入進度

## 非功能需求
- 效能：機場搜尋 < 200ms（本地）；機票搜尋 < 15 秒（真實）/ < 3 秒（模擬）
- 資料品質：真實資料需標示來源和時間戳；模擬資料需明確標示
- 資料量：全球機場資料 ~500KB（壓縮後），可接受首次載入
- 相容性：機場搜尋支援中文和英文輸入
- 快取：真實資料 30 分鐘快取；機場資料每月更新

## 安全性需求
- 爬蟲不得違反航空公司的 robots.txt
- 爬蟲請求頻率需控制，避免被封鎖
- 不儲存使用者的信用卡或支付資訊

## 成功指標
- 機場搜尋：使用者平均 2 次按鍵內找到目標機場
- 機票搜尋：真實資料命中率 > 80%（無 fallback 到模擬資料）
- 外站票比較：30%+ 的使用者在搜尋來回票後會查看外站票方案
