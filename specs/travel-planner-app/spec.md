## 需求摘要

一個針對家庭旅遊和團體旅遊設計的 App，讓使用者可以共同規劃行程、編輯修改、最終定案，並提供花費預估、簡易記帳與拆帳功能，以及行程分享。特別強調的第一優先功能是「外站票（四段票）查找」——幫助使用者快速找到比直飛便宜的機票組合。

---

## 使用者故事

### 機票搜尋者（Ticket Searcher）— 🎯 第一優先功能

#### US-1：外站票（四段票）快速查找
- **作為**機票搜尋者，**我想要**輸入出發地和目的地後，系統自動搜尋並列出所有可能的外站票（四段票）組合和價格，**以便**我能一眼看出哪個方案最省錢
- **驗收條件**：
  - [ ] 使用者輸入出發地、目的地、出發日期和回程日期後，系統顯示外站票組合列表
  - [ ] 每個組合清楚顯示：四段航程（去程外站→出發地→目的地、回程目的地→出發地→外站）、各段航班資訊、總價格
  - [ ] 可以和直飛票價做比較，顯示價差金額和省下的百分比
  - [ ] 結果可依價格、轉機時間、航空公司排序
  - [ ] 支援多人同行搜尋（輸入人數，顯示總價）

#### US-2：外站票概念說明
- **作為**不熟悉外站票的旅伴，**我想要**系統用簡單易懂的方式解釋什麼是外站票/四段票，**以便**我能理解為什麼這樣飛比較便宜
- **驗收條件**：
  - [ ] 首次使用時顯示外站票概念的圖示說明
  - [ ] 搜尋結果中每個組合都有視覺化的航線圖
  - [ ] 提供「為什麼比較便宜？」的簡單解釋連結

---

### 旅遊發起人（Trip Organizer）

#### US-3：建立旅遊計畫
- **作為**旅遊發起人，**我想要**快速建立一個新的旅遊計畫並設定基本資訊（目的地、日期、預算），**以便**開始規劃行程
- **驗收條件**：
  - [ ] 可以建立新旅遊計畫，填入名稱、目的地、日期範圍、預估預算
  - [ ] 建立後自動生成一個可分享的邀請連結
  - [ ] 可以設定計畫為「規劃中」或「已定案」狀態

#### US-4：規劃每日行程
- **作為**旅遊發起人，**我想要**在每一天加入景點、餐廳、交通方式等行程項目，**以便**建立完整的每日行程表
- **驗收條件**：
  - [ ] 每天可以新增多個行程項目（景點、餐廳、交通、住宿、其他）
  - [ ] 每個項目可填入名稱、時間、地點、備註、預估花費
  - [ ] 行程項目可拖曳排序
  - [ ] 可以查看整趟旅遊的行程總覽（時間軸或日曆視圖）

#### US-5：邀請旅伴加入
- **作為**旅遊發起人，**我想要**透過連結或 App 內邀請旅伴加入計畫，**以便**大家一起編輯行程
- **驗收條件**：
  - [ ] 可以產生邀請連結，分享到 LINE、WhatsApp 等通訊軟體
  - [ ] 旅伴點連結後可以直接加入計畫（不需要先註冊也能查看）
  - [ ] 可以設定旅伴的權限（可編輯 / 僅查看）

#### US-6：行程定案
- **作為**旅遊發起人，**我想要**在大家討論完畢後將行程標記為「已定案」，**以便**所有人都知道這是最終版本
- **驗收條件**：
  - [ ] 可以將計畫狀態從「規劃中」改為「已定案」
  - [ ] 定案後發送通知給所有旅伴
  - [ ] 定案後行程預設鎖定，需解鎖才能修改
  - [ ] 保留定案前的編輯歷史

---

### 旅伴 / 團員（Trip Member）

#### US-7：共同編輯行程
- **作為**旅伴，**我想要**在行程上新增建議、修改項目或留言討論，**以便**參與行程規劃
- **驗收條件**：
  - [ ] 可以新增、修改、刪除行程項目（在有編輯權限時）
  - [ ] 可以對任何行程項目留言討論
  - [ ] 多人同時編輯時不會互相覆蓋（即時同步或衝突提示）
  - [ ] 可以看到誰做了什麼修改（編輯紀錄）

#### US-8：查看與分享行程
- **作為**旅伴，**我想要**以清楚的格式查看定案後的行程，並分享給沒有使用 App 的人，**以便**所有相關人員都能掌握行程資訊
- **驗收條件**：
  - [ ] 行程可以一鍵匯出為圖片或 PDF
  - [ ] 可以產生唯讀分享連結（不需要下載 App 就能看）
  - [ ] 分享的行程包含每日行程、地點、時間、備註

---

### 記帳管理者（Expense Manager）

#### US-9：花費預估
- **作為**記帳管理者，**我想要**根據行程項目自動匯總預估花費，**以便**掌握整趟旅遊的預算狀況
- **驗收條件**：
  - [ ] 自動加總所有行程項目的預估花費
  - [ ] 顯示預估總花費 vs 設定的預算，提醒是否超支
  - [ ] 可依類別（交通、住宿、餐飲、門票、其他）查看花費分佈
  - [ ] 支援多幣別，自動換算

#### US-10：簡易記帳
- **作為**記帳管理者，**我想要**在旅遊過程中快速記錄每筆實際花費，**以便**追蹤實際支出
- **驗收條件**：
  - [ ] 可以快速新增一筆花費（金額、類別、備註、付款人）
  - [ ] 可以標記哪些人分攤這筆費用
  - [ ] 可以拍照收據附加到花費紀錄
  - [ ] 支援多幣別輸入

#### US-11：拆帳結算
- **作為**記帳管理者，**我想要**系統自動計算每個人應付和應收的金額，**以便**旅遊結束後快速結清費用
- **驗收條件**：
  - [ ] 支援多種拆帳方式：均分、按比例、自訂金額
  - [ ] 自動計算最簡化的轉帳方案（最少轉帳次數）
  - [ ] 產出結算報告，清楚顯示誰要付誰多少錢
  - [ ] 可以標記已結清的款項
  - [ ] 可以將結算結果分享給所有旅伴

## 衝突標記

- ⚠️ 機票搜尋者想要最便宜的外站票（可能轉機時間長），但旅伴希望行程舒適不要轉太多次 → 需要在搜尋結果中同時顯示價格和轉機時間，讓使用者自行權衡
- ⚠️ 旅遊發起人想快速定案，但旅伴可能需要更多討論時間 → 需要提供投票或確認機制
- ⚠️ 記帳管理者需要精確拆帳，但部分旅伴可能覺得記帳麻煩 → 記帳操作要盡量簡化

## 非功能需求
- 效能：機票搜尋結果應在 10 秒內返回；行程共同編輯需即時同步（延遲 < 2 秒）
- 安全性：用戶資料加密儲存；邀請連結需設定有效期限
- 相容性：支援 iOS 和 Android；分享連結需支援各大瀏覽器
- 離線支援：已下載的行程在無網路時仍可查看
- 語言：介面支援繁體中文
