## 需求摘要

四項核心體驗補強：(1) 移除所有 mock/模擬資料，API 失敗時顯示錯誤而非假資料；(2) 前端行程規劃和記帳拆帳頁面接上已完成的後端 API；(3) 機票搜尋結果一鍵加入旅遊行程；(4) 追蹤航線可設定目標價格，票價低於目標時提醒。

---

## 使用者故事

### US-17：移除 mock 資料（清理）
- **作為**機票搜尋者，**我想要**看到的結果都是真實資料或明確的錯誤提示，**以便**不被假資料誤導
- **驗收條件**：
  - [ ] 移除前端所有 mock/simulated fallback 邏輯
  - [ ] API 呼叫失敗時顯示「查詢失敗，請稍後再試」錯誤訊息
  - [ ] 後端 SimulatedCrawler 從 CrawlerRouter 中移除（只保留真實爬蟲）
  - [ ] 搜尋結果頁面不再出現 source: "simulated" 的結果
  - [ ] 外站票搜尋同樣移除 mock fallback

---

### US-18：行程規劃前端頁面
- **作為**旅遊發起人，**我想要**在 App 上建立旅遊計畫、新增行程、邀請旅伴、定案，**以便**不用只靠 API 測試工具操作
- **驗收條件**：
  - [ ] 「我的旅程」列表頁：顯示所有計畫（名稱、目的地、日期、狀態、成員數）
  - [ ] 建立新計畫表單：名稱、目的地、日期、預算
  - [ ] 計畫詳情頁：每日行程列表，可新增/編輯/刪除項目
  - [ ] 每個行程項目顯示：名稱、時間、地點、預估花費、類型圖示
  - [ ] 邀請旅伴：產生連結 + 複製按鈕
  - [ ] 定案流程：發起定案 → 顯示成員確認狀態
  - [ ] 成員列表顯示角色（發起人/編輯者/查看者）

### US-19：記帳拆帳前端頁面
- **作為**記帳管理者，**我想要**在 App 上快速記帳和查看拆帳結算，**以便**旅遊中隨時記錄花費
- **驗收條件**：
  - [ ] 快速記帳表單：金額（大字輸入）+ 付款人 + 類別（可選）+ 備註（可選）
  - [ ] 花費列表：按時間排序，顯示金額、付款人、類別
  - [ ] 預算進度條：預估 + 實際 vs 預算，超支時變紅
  - [ ] 按類別查看花費分佈（簡易橫條圖）
  - [ ] 拆帳結算頁面：誰要付誰多少錢，可標記已結清
  - [ ] 結算結果可一鍵複製文字分享

---

### US-20：機票搜尋 → 行程一鍵整合
- **作為**機票搜尋者，**我想要**搜到好的航班後，一鍵把它加入我的旅遊行程，**以便**不用手動記錄航班資訊
- **驗收條件**：
  - [ ] 每張航班結果卡片上顯示「+ 加入行程」按鈕
  - [ ] 點擊後彈出選擇計畫的下拉（或建立新計畫）
  - [ ] 選擇後自動建立行程項目：類型=交通、名稱=航空公司+航班號、時間=起飛時間、預估花費=票價
  - [ ] 加入成功後顯示提示「已加入 [計畫名稱]」
  - [ ] 外站票結果也支援一鍵加入（加入 4 段航程）

### US-21：票價監控與提醒
- **作為**機票搜尋者，**我想要**設定目標價格，票價低於目標時收到提醒，**以便**在最便宜的時候購買
- **驗收條件**：
  - [ ] 追蹤航線可設定目標價格（例如：TPE→NRT 低於 8,000 元時提醒）
  - [ ] 追蹤列表顯示：當前最低價、目標價、差距
  - [ ] 每日爬取後比對目標價，低於目標的航線標記 🔔
  - [ ] 搜尋結果中如果票價為歷史最低，顯示「🏷️ 歷史新低！」標籤
  - [ ] 追蹤頁面的提醒航線置頂 + 醒目標示

## 衝突標記

- ⚠️ 移除 mock 後，如果真實爬蟲也失敗，使用者會看到空結果 → 需要清楚的錯誤提示 + 建議
- ⚠️ 票價提醒目前只能在 App 內查看（沒有推播） → 第一版先做 App 內提醒，推播通知後續再加

## 非功能需求
- 效能：行程頁面載入 < 2 秒；記帳輸入 < 2 秒完成一筆
- UX：記帳金額使用大字鍵盤輸入（手機優化）；行程項目支援上下拖曳排序
- 相容性：Web 版和 React Native 版一致的 UI 風格

## 安全性需求
- 行程資料只有成員可見
- 記帳資料只有旅遊計畫成員可存取

## 成功指標
- 移除 mock 後零模擬資料出現
- 行程建立到定案的完成率 > 50%
- 記帳操作平均 < 5 秒
- 30%+ 的搜尋使用者至少一次「加入行程」
- 設定票價提醒的使用者 > 20%
