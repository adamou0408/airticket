# 衝突編號：CONFLICT-004

## 狀態：`detected`

## 衝突描述
- **機票搜尋者** 需要：看到真實的航空公司票價，才能做出正確的購票決策
- **系統穩定性** 需要：航空公司官網有反爬蟲機制，真實資料可能不穩定、查詢時間長
- **為什麼衝突**：使用者期望看到真實票價，但從航空公司官網爬取資料有被封鎖的風險。如果爬取失敗，使用者會看到模擬資料，可能造成期望落差。

## AI 分析
這是**資料可靠性 vs 資料真實性**的衝突。核心問題是如何在「一定有資料可看」和「資料是真實的」之間取得平衡。

可能的方向：
- 串接第三方 API（Amadeus、Skyscanner）作為主要資料源，穩定度高
- 自行爬蟲作為補充，爬取失敗時 fallback 到第三方 API
- 明確標示資料來源（真實/模擬），讓使用者知道資料的可信度

## 解決方案
- [ ] 方案 1：（真實優先）只使用爬蟲取得真實資料，無資料時顯示「暫無報價」
- [ ] 方案 2：（穩定優先）串接第三方機票 API（如 Amadeus）作為主要來源，穩定且合法
- [ ] 方案 3：（混合方案）先嘗試真實資料 → 失敗時 fallback 到模擬資料，清楚標示資料來源
