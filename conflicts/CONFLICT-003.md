# 衝突編號：CONFLICT-003

## 狀態：`detected`

## 衝突描述
- **記帳管理者** 需要：每筆花費都被精確記錄，才能準確拆帳、避免事後爭議
- **旅伴（團員）** 需要：不想在旅遊過程中花太多時間記帳，覺得麻煩會影響旅遊體驗
- **為什麼衝突**：精確記帳需要及時、詳細的輸入，但旅遊中大家都想放鬆，不想一直拿手機記帳。記太少會拆帳不準，記太多會影響旅遊心情。

## AI 分析
這是典型的**UX 衝突**（精確度 vs 便利性）。核心問題在於如何讓記帳動作極度簡化，降低旅伴的抗拒感，同時又能收集到足夠的拆帳資訊。

可能的方向：
- 極簡記帳：只需輸入金額和誰付的，其他資訊選填
- 快速記帳捷徑：常用的花費類型（餐飲、交通、門票）一鍵選擇
- 允許拍照收據，事後再補細節
- 「大家先均分」作為預設，只有特殊情況才需要調整

## 解決方案
- [ ] 方案 1：（優先滿足記帳管理者）每筆花費要求填寫完整資訊才能儲存
- [ ] 方案 2：（優先滿足旅伴）最簡模式：只需金額 + 付款人，其他全部選填或預設均分
- [ ] 方案 3：（折衷方案）提供「快速記帳」（金額 + 付款人，2 秒完成）和「詳細記帳」兩種模式；預設使用快速模式，記帳管理者可事後補充細節
