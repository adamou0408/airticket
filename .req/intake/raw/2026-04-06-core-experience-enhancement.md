# 💡 需求補強 — 核心體驗三合一

**我是誰 / Who am I**：專案發起人（Amber 的先生）

**我想要 / I want**：

## 1. 機票搜尋 → 行程規劃一鍵整合
搜到便宜的機票後，可以一鍵把這個航班加入我的旅遊計畫行程中。不用自己記住價格再手動輸入。
- 搜尋結果的每張航班卡片上加一個「加入行程」按鈕
- 點了之後選擇要加入哪個旅遊計畫（或建立新的）
- 自動填入航班資訊（航空公司、時間、價格）到行程項目

## 2. 票價監控與提醒
我追蹤了航線，但不知道票價什麼時候降了。希望可以設定「當票價低於 X 元時通知我」。
- 在追蹤航線旁邊可以設定目標價格
- 每日爬取時如果票價低於目標，標記提醒
- 搜尋結果如果比歷史最低價還低，顯示「歷史新低！」標籤

## 3. 前端行程規劃 + 記帳拆帳頁面
後端 API 都做好了但前端是空的 placeholder，要把它們接起來：
- 行程頁面：建立計畫、新增/編輯行程項目、邀請旅伴、定案流程
- 記帳頁面：快速記帳、花費列表、預算進度條、拆帳結算顯示

## 4. 移除 mock 資料
所有搜尋結果都要走真實 API，不再 fallback 到假資料。如果 API 失敗就顯示「查詢失敗，請稍後再試」。

**為什麼 / Why**：
1. 機票和行程是分開的兩個功能，不串起來使用者要手動記錄很不方便
2. 追蹤航線的核心價值就是「便宜的時候通知我」，沒有這個追蹤功能只做了一半
3. 行程和記帳是旅遊規劃的核心功能，不能只有後端沒有前端
4. mock 資料會誤導使用者，以為看到的是真實票價

**補充 / Additional notes**：
- 這四個改動有依賴關係：先移除 mock (#4) → 再做行程前端 (#3) → 然後串聯機票到行程 (#1) → 最後加票價提醒 (#2)
- 行程和記帳前端可參考現有的 Web 版風格（卡片式、橘色主題）
